Test 1: Unexpected Hanging Paradox

Challenge Level: Basic | Domain: Epistemic Modal Logic & Game Theory

Complete Test Prompt:
In the Unexpected Hanging Paradox variant: A judge tells a prisoner he will be executed at noon on a weekday next week, but the execution will be a surprise (he won't know it's coming that morning). The prisoner reasons: 'It can't be Friday, because if Thursday noon passes, I'd know it must be Friday. By similar logic, it can't be Thursday...' and concludes he cannot be executed. Yet he is surprised when executed on Wednesday.

Task: Analyze this paradox using epistemic modal logic. What's the flaw in the prisoner's reasoning? Formalize the concepts of 'knowledge' and 'surprise' that makes the judge's statement both coherent and executable.
Complete o3 Response - Epistemic Model Evolution Framework
o3
OpenAI o3 - Dynamic Epistemic Logic Analysis

Formal Framework

Epistemic Operators
$K_n \varphi = \text{"On the morning of day } n \text{ the prisoner knows } \varphi\text{"}$
$E = \text{"One single execution takes place next week"}$
$D = \{Mon, Tue, Wed, Thu, Fri\}; \quad d^* \in D \text{ is the unique execution day}$
$\text{Surprise requirement: } \neg K_n(d^*=n) \text{ for the actual day } n$

Judge's Promise Formalization

$P := E \land \bigvee_{d \in D}[(d^*=d) \land \neg K_d(d^*=d)]$

Translation: "Exactly one day is the execution and, on that day's morning, the prisoner will not know it is today."

Epistemic Model Evolution

  • Initial state $(M_0)$: Five possible worlds, one for each execution day
  • Morning updates: Public announcement of "not executed before $n$"
  • Model progression: $(M_0 \mid \text{not-before-Mon}) \mid \ldots \mid (\text{not-before-}n)$

Core Insight: Where Backward Induction Fails

The prisoner's reasoning requires:

$K_4((\text{only Friday remains}) \to K_4(d^*=\text{Fri}))$

This is a second-order knowledge statement about future knowledge. The critical flaw:

  • S5 knowledge is factive and introspective about current knowledge
  • It does NOT guarantee knowledge preservation under future updates
  • Dynamic epistemic logic shows knowledge is not preserved under arbitrary public announcements

Resolution

Choose Wednesday as $d^*$. Morning evaluations:

  • Monday: $\{Mon,Tue,Wed,Thu,Fri\}$ possible $\Rightarrow \neg K_1(d^*=Mon)$ ‚úì
  • Tuesday: $\{Tue,Wed,Thu,Fri\}$ possible $\Rightarrow \neg K_2(d^*=Tue)$ ‚úì
  • Wednesday: $\{Wed,Thu,Fri\}$ possible $\Rightarrow \neg K_3(d^*=Wed)$ ‚úì

The judge can execute on Wednesday with genuine surprise.

Key Takeaways

  1. Surprise = "not known that morning" $(\neg K_n(d^*=n))$
  2. Backward induction illegitimately assumes second-order knowledge about future states
  3. Dynamic epistemic logic cleanly separates current from hypothetical post-announcement knowledge

  • Thursday eliminated: With Friday gone, $K_{m(Thu)} E_{Thu}$ would hold

The Prisoner's Modal Flaw

The prisoner commits an illicit cross-time closure:

  • Assumes: $K_{t_0}(\varphi \to K_{m(d)} \psi)$ implies $K_{t_0}(\varphi \to \psi)$
  • Treats $J$ as persistent knowledge reusable at all future times

  2. No cross-time axiom: $K_t \varphi$ doesn't imply $K_s \varphi$ for $s > t$
  3. Dynamic instability: Truth of $\neg K_{m(d)} E_d$ can flip with observations

Resolution Coherence

  • $J$ evaluated at $t_0$ requires existence of executable surprise day
  • Wednesday execution: $Surprise(Wed)$ true at $m(Wed)$ (Wed/Thu both live)
  • No contradiction: $J$ need not remain true at all times/counterfactuals
